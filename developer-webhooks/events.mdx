## Event names

The following `event` strings are valid when subscribing. The spelling `audince.*` matches the product code.

| `event` | When it fires (high level) |
|---------|----------------------------|
| `message.received` | Inbound WhatsApp message received |
| `message.sent` | Outbound message send completed (Meta response shape in `data`) |
| `message.status.update` | Message delivery/read status update |
| `contact.created` | Contact created |
| `contact.updated` | Contact updated |
| `contact.deleted` | Contact deleted |
| `audince.created` | Contact group (audience) created |
| `audince.updated` | Contact group updated |
| `audince.deleted` | Contact group deleted |
| `autoreply.created` | Basic (canned) reply created |
| `autoreply.updated` | Basic reply updated |
| `autoreply.deleted` | Basic reply deleted |
| `label.created` | Label created |
| `label.updated` | Label updated |
| `label.deleted` | Label deleted |

<Note>
  All JSON examples below use **redacted placeholders** (`xxxx`, masked IDs, and generic timestamps). Real payloads contain your customers’ data—handle them accordingly under your privacy policy.
</Note>

## Example payloads

These mirror the structure produced for testing in the product; **live** `data` may include more or fewer fields over time. Branch on `event` and ignore unknown keys.

### message.received

```json
{
  "event": "message.received",
  "data": {
    "value": {
      "messaging_product": "whatsapp",
      "metadata": {
        "display_phone_number": "xxxxxxxxxxxx",
        "phone_number_id": "xxxxxxxxxxxxxxxx"
      },
      "contacts": [
        {
          "profile": { "name": "xxxx" },
          "wa_id": "+xxxxxxxxxxxx"
        }
      ],
      "messages": [
        {
          "from": "+xxxxxxxxxxxx",
          "id": "wamid.xxxx",
          "timestamp": "1700000000",
          "text": { "body": "xxxx" },
          "type": "text"
        }
      ]
    },
    "field": "messages"
  }
}
```

### message.sent

```json
{
  "event": "message.sent",
  "data": {
    "success": true,
    "data": {
      "messaging_product": "whatsapp",
      "contacts": [{ "input": "+xxxxxxxxxxxx", "wa_id": "+xxxxxxxxxxxx" }],
      "messages": [{ "id": "wamid.xxxx", "message_status": "accepted" }]
    }
  }
}
```

### message.status.update

```json
{
  "event": "message.status.update",
  "data": {
    "value": {
      "messaging_product": "whatsapp",
      "metadata": {
        "display_phone_number": "+xxxxxxxxxxxx",
        "phone_number_id": "xxxxxxxxxxxxxxxx"
      },
      "statuses": [
        {
          "id": "wamid.xxxx",
          "status": "delivered",
          "timestamp": "1700000000",
          "recipient_id": "xxxxxxxxxxxx"
        }
      ]
    },
    "field": "messages"
  }
}
```

### autoreply.created

`metadata` is often a **string** containing JSON (text/image configuration). Inner text is redacted below.

```json
{
  "event": "autoreply.created",
  "data": {
    "name": "xxxx",
    "trigger": "xxxx",
    "match_criteria": "contains",

    "created_at": "2025-01-01T12:00:00.000Z",

  }
}
```
